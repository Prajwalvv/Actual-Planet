API Documentation

Everything you need to integrate real-time swarm intelligence into your application.

Rylvo exposes a REST API that lets you query a live colony of autonomous agents. Each query deploys thousands of specialized ants that discover, validate, and cross-reference data from multiple sources — then returns structured intelligence with conviction scores.

Base URL
https://api.rylvo.dev/api

Authentication

All requests require an API key. Include it in the Authorization header:

Authorization: Bearer sk_live_your_api_key_here

Generate your API key from the Developer Console. Keep it secret — it grants full access to your account's credits.

Quick Start

Query the swarm in one API call. Runtime is query-driven: the engine auto-starts for /api/query and auto-stops after 120s idle.

// Node.js / TypeScript const res = await fetch('https://api.rylvo.dev/api/query', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${process.env.SWARM_API_KEY}` }, body: JSON.stringify({ query: "What entities are showing unusual activity?", model: "discover" }) }); const data = await res.json(); console.log(data.data.topReports);

Python

import requests res = requests.post( "https://api.rylvo.dev/api/query", headers={"Authorization": f"Bearer {API_KEY}"}, json={"query": "What's trending right now?", "model": "full"} ) for report in res.json()["data"]["topReports"]: print(f"{report['locationId']}: {report['polarity']} ({report['conviction']:.2f})")

Models

Each model biases the adaptive planner toward different discovery/corroboration strategies. Choose based on your use case:

ModelCreditsBest for
discover1Broad exploration, trending entities, new sources
precise3Deep, cross-validated analysis on specific entities
correlate3Cross-entity pattern detection, convergences
sentiment1Social signals, crowd mood, opinion mining
full5All castes at max. Discovery + precision + correlation

Model Wait Limits

timeout is validated per model. If timeout is out of range, the API returns 400 with model-specific bounds.

POST /api/query

The primary endpoint. Query the swarm with natural language and/or specific topics/entities. Runtime is query-driven with per-model shard pools: shards auto-start on demand, route by load, and auto-stop after idle.

POST/api/query
Returns topic resolutions, evidence, execution planning, source coverage, and runtime metadata for your query.
ParameterTypeDescription
querystringNatural language query. E.g. "What's showing unusual activity?"
symbolsstring[]Specific topics/entities/phrases to resolve. E.g. ["smart drugs", "India", "AI chip export controls"]
modelstringSwarm model to use for this query. If omitted: running model (if allowed) else plan default (discover on free, full on paid plans).
depthstringquick | standard | deep
timeoutnumberHow long the client can wait (seconds). Must be within model-specific min/max bounds.
streambooleanIf true or Accept: text/event-stream is used, the server emits lifecycle/evidence SSE events until final output.
terrainHintsstring[]Optional terrain planner hints: news, forum, docs, academic, company, general-web, social-signal.

Query Response Contract

POST /api/query returns structured topic resolutions, evidence, execution planning, and coverage metadata so LLM clients can reason about value even when exact symbols are missing.

FieldDescription
data.topicResolutions[]Primary per-input topic resolution with status in exact | phrase_match | alias_match | fuzzy_match | related_only | not_found, plus evidence ids, alternatives, and next actions.
data.evidence[]Normalized evidence extracted from public-web sources. Includes domain, terrain, entities, phrases, claims, freshness, and confidence.
data.executionPlanTerrain plan, provider allocation, breed mix, and phase budgets chosen for this query.
data.sourceCoverageHow much of the public web was covered: evidence count, domain count, blocked URLs, provider hits, terrain hits, and promoted topics.
data.queryMeta.coverageCoverage KPI block: requestedSymbols, exactCount, resolvedCount, relatedOnlyCount, notFoundCount, coverageRatio, usefulnessScore.
data.queryMetaRuntime metadata: model requested/used, timeout requested/accepted/bounds, queue position/wait, auto-start/cold-start, elapsed time, ISO responded timestamp, partial flag, corroboration totals, and streaming stats.
{
"ok": true,
"data": {
  "topicResolutions": [
    { "input": "smart drugs", "status": "related_only", "alternatives": [...] }
  ],
  "evidence": [ { "domain": "news.ycombinator.com", "terrain": "forum" } ],
  "executionPlan": { "terrains": [...], "breedPlan": [...] },
  "queryMeta": {
    "modelUsed": "discover",
    "queuePosition": 1,

GET /api/analyze/:locationId

Deep analysis on a single entity. Returns a full IntelligenceReport with signals, correlations, and quality assessment.

GET/api/analyze/Tesla
Returns polarity, conviction, confidence, signals, correlations, data quality for one entity.

GET /api/discover

GET/api/discover?limit=20
Returns recently discovered entities and their latest reports, sorted by activity.

GET /api/reports

GET/api/reports?polarity=negative&minConviction=0.5&limit=10
Returns all intelligence reports. Filterable by polarity, minimum conviction, and limit.

GET /api/correlate

GET/api/correlate
Returns cross-entity correlations: convergences, divergences, cascades detected by the swarm.

GET /api/health

GET/api/health
Colony health report: health score, signal-to-noise ratio, stale locations, active locations.

WebSocket Stream

Connect to the real-time event stream for live updates:

const ws = new WebSocket('wss://api.rylvo.dev/ws'); ws.onmessage = (event) => { const msg = JSON.parse(event.data); // msg.type: 'tick' | 'discovery' | 'emergence' | 'info' | 'error' console.log(msg.type, msg.data); };

LLM Tool Calling

Register swarm_query as a tool in your LLM to give it real-time awareness:

// OpenAI function calling format { "name": "swarm_query", "description": "Query the open-web adaptive swarm for evidence-ranked topic intelligence under a strict timeout budget", "parameters": { "type": "object", "properties": { "query": { "type": "string" }, "symbols": { "type": "array", "items": { "type": "string" } }, "model": { "type": "string", "enum": ["discover","precise","correlate","sentiment","full"] }, "timeout": { "type": "number", "description": "Client wait budget in seconds (model-specific bounds)" }, "stream": { "type": "boolean" }, "terrainHints": { "type": "array", "items": { "type": "string" } } } } }

Fetch the full schema programmatically: GET /api/tool-definition

Error Codes

StatusMeaning
200Success
400Bad request — missing or invalid parameters
401Unauthorized — invalid or missing API key
404Entity not found — swarm hasn't discovered this entity yet
429Rate limited, out of credits, or runtime backpressure (queue saturation with retry hint)
503Runtime unavailable for non-query endpoints (engine will auto-start on next /api/query)

All error responses follow the format:

{ "ok": false, "error": "Description of what went wrong" }
