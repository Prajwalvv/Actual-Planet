What is this? Runtime is fully query-driven. There are no manual start/stop controls. The first query starts model shards automatically, local FIFO is per shard, and idle shards auto-stop after timeout.
  • Model selection now belongs to each /api/query request.
  • Timeout is validated per model (min/default/max).
  • Queue is deterministic per shard with global backpressure.
  • Symbol fallback returns structured resolution states when exact symbols are missing.
  • Check WebSocket Monitor for runtime events in real time.

Runtime Policy

Lifecycle: query-driven (auto-start per query model)
Queue: per-model shard pools with local FIFO and QoS admission control
Idle Auto-stop: 120 seconds
Model wait limits: discover/correlate/full 20-120s (default 45s), precise 15-90s (default 35s), sentiment 10-60s (default 25s)

LLM Simulator

Simulate what an LLM sees when it calls POST /api/query. Test different queries, entities, depths.

What is this? This simulates an LLM calling the Rylvo API. When an LLM uses the swarm_query tool, it sends a query here and gets back intelligence data.
  • Topic resolutions show whether the swarm found an exact topic, phrase match, fuzzy match, related-only result, or nothing useful inside the time budget.
  • Entities field — explicit topics or phrases to resolve. Leave blank to query by keywords only.
  • Model — each query can pick a model. Runtime will switch automatically if needed.
  • Timeout — client wait budget, validated per model (min/default/max).
  • Streaming — optional SSE mode for lifecycle events and live evidence updates.
  • Depth — quick (less data, faster), standard, deep (more data, slower)
  • The Tool Definition panel shows the OpenAI function-calling schema that LLMs see.

Ant Inspector

Live view of all ant castes — counts, locations, deposits, performance breakdown.

What is this? Shows every type of ant in the colony and what they're doing. Each caste has a specific job:
  • Scouts — crawl the web discovering new entities (the "explorers")
  • Foragers — collect live data (value, activity, sentiment) for discovered entities
  • Amplifiers — boost strong signals via positive feedback loops
  • Correlators — compare entities, detect convergence/divergence patterns
  • Validators — cross-check signals for consistency and flag anomalies
  • Guards — detect echo chambers and quarantine bad data
  • Harvesters — turn raw pheromone data into intelligence reports
  • Nurse — monitors colony health, detects stale/saturated locations

WebSocket Monitor

Real-time stream of adaptive runtime events. See planning, discovery, emergence, and finalization as they happen.

What is this? A live event stream showing exactly what the adaptive runtime is doing right now. Click CONNECT to start watching.
  • tick — one adaptive query cycle completed
  • discovery — providers or breeds found new public-web sources or evidence
  • emergence — planner, resolver, or corroboration state advanced during the query window
  • info — finalization or lifecycle updates
  • error — something went wrong (blocked by website, timeout, etc.)
This is the best way to see whether the runtime is actually producing useful evidence during a query window.
